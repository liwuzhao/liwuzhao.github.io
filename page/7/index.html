提取练习— 「根据购物网站第一周」

发表于 2017-05-05

今天实作购物网站第一周「商家用户故事」 的课程，发现其实和job-listing前面的很相像，就当做了一次提取练习。

基础部分

安装bootstrap

  • gem 'bootstrap-sass'

  • application.css => application.scss

  • application.scss 中加入

安装simple_form

  • gem 'simple_form
  • rails g simple_form:install —bootstrap

安装devise

  • gem 'devise
  • rails g devise:install
  • rails g devise user
  • rails db:migrate

admin的CRUD

  • rails g controller admin::products

  • routes中加入

  • 建立model，建立时直接添加需要的栏位

  • CRUD和对应的view

admin 可以登录后台

  • 新增一个is_admin的栏位
    rails g migration add_is_admin_to_user
  • 在user中加入admin？的判断式

  • 建立admin判断,在application_controller.rb

  • 限制只有admin才能进行admin下的CRUD
    before_action :admin_required

【总结】用params和where抓出工作分类

发表于 2017-04-24

很多同学都加入对工作进行分类的这个栏位，那么如何实现查看每个分类的工作呢？
以我的自己专案分享一下思路

##第一步

加入一个导向设计分类的按钮，括号中的内容意思是把设计这个value赋给category这个key

##第二步
在controller中捞出每个分类的工作

观察url，当点击第一步的link时，地址是这样的
http://localhost:3000/jobs?category=设计
当url有category这个key时，用params把这个key对应的value，即设计赋给@category
然后用where 把category栏位(注意这里代表的是栏位，是你建分类的时候的栏位名称)中带有设计这个value的的jobs捞出来

上面就是大致的思路，具体的代码可以根据个人专案进行修改
