ORID 5.26 Debug + 第二版

发表于 2017-05-26

Objective

完成了魔改第二版

Reflective

Interpretive

  • 在商品详情页修改加入购物车的商品数量(明天对这个做好总结)

  • Deploy到heroku时候出现一个报错

    提示是PGError: ERROR: column “name” of relation “users” already exists 栏位有重复，但是本地上是没问题的，尝试把name栏位名字改了，又出现了另外一个栏位的重复

    最后把数据库清掉，不过用的指令是

    heroku pg:reset DATABASE 参考这里

  • 测试出很多bug，一一做了修复，虽然有点些枯燥，但是还是能自己解决，总结几点经常出错的

    1. create或者update不成功时，看看controller里面有没有给他们相应栏位的参数 ，看xxx_params
    1. 很多功能中对current_user存在的情况做了判断，如果有!cuurent_user的话很容易报错，要以游客身份去测试一下网站
    1. table和column多的话，很容易出错，多点看schema，明确每个table的有什么栏位，在哪里用到

【总结】在购物车中使用 `+`和 `- `按钮改变购物数量

发表于 2017-05-25

给购物车添加 + 和 - 的按钮直接修改购买商品的数量

我的思路是这样的

添加两个这两个按钮

定义加和减的action

第一步 在购物车加上这两个按钮

第二步 添加这两个action

这两个action的思路是，对cart_item中product的quantity进行修改

第三步 修改routes

添加了两个新的action，我们得给它一个地址

效果
这样就可以用加减按钮去修改购物车中商品的数量了，购物总价也会实时更新。具体的美化修饰，大家可以自行修改。

Ps:但是如果数量要直接从1改成100，除了教材中select的方法，我还没有找到更好的解决的方法。

5.17 学习利用碎片时间

发表于 2017-05-17

Objective

今天没有完整的时间，利用碎片时间做了一点事情

Reflective

遇到问题时会焦虑，有时候甚至会把问题夸大

Interpretive

一般周三我是很难找到完整的一段时间的，比如晚上开始学习都已经十一点了，今天走在路上我思考了一下这种情况要怎么办，然后想出了几个利用碎片时间的办法。

  • 在slack上看看同学的提问和助教的解答，了解一下自己没有遇到过的bug，自己想想debug的思路，再看看助教是怎么解决的

  • 看看同学的logdown，经常会看到一些干货分享。比如今天我看到了好几个同学都记录到自己或者队友写了完整的userstory，又提醒了自己，把自己零散记录的想法写成userstory，因为已经有记录，只用整理，花不了什么时间。还有个收获是看到有同学参考了我的分享，挺高兴的，于是跟着文志鹏师兄的分享给logdown开了个评论功能。

  • 在脱产群中经常有技术性问题的讨论，白天就那么看过去了，晚上回来可以认真从头看一看，有适合自己的也可以尝试一下或者记录下来

  • 碎片时间里也可以想想教程、知识点笔记、对教程的复盘笔记，又或者想想自己之前想要做的一些功能的开发思路是怎么样的。突然GET到锤子的“闪念胶囊” ，真的是神器啊，好想用啊！

  • 做一些用时不用那么长的事情，比如读一篇英文文章。

  • 总的来说，这些都是些用时短、不太费力的功夫

还有两个重要领悟是

  • 学习一定要跳出舒适区，切忌做一个假装勤奋者，只挑些不用思考的事情来做

  • 如果遇到情绪低落的情况，一定要及时想通，把自己拉出来（想通这个过程也能GET到不少新领悟）

Decisional

明天把切换订单状态的教程做完
