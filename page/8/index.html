建立model时写错字母的解决方法

发表于 2017-04-13

昨天在做的时候遇到了这个问题，只能自己一个一个的全部改回来，非常麻烦，今天看到论坛上有人分享了解决方法，复制到自己的logdown上，以便以后出错时查阅。

#解决方案一（只执行了一步rake db：migrate）
第一步：
前往修改db/xxxx_create_groups.rb修改错误的输入，然后保存。

第二步：
在终端输入
rake db:rollback （这条命令的意思是：将数据库后退一步）

第三部：
rake db:migrate (重新将数据库跑起来，这时候你的title已经被修改为正确的了)

#解决方案二（如何你执行了很多命令）
第一步：
新建一个修改的db
rails g migration renamecolumn (其中“renamecolumn”可以自己更换为其他名称)

第二步：
修改db/xxxxx.renamecolumn.rb

class Renamecolumn < ActiveRecord::Migration[5.0]
def change
rename_column:post, :titlr, :title
end
end

第三步：
rake db:migrate

关于model，用resume举例

  • rails g model resume job_id:integer user_id:integer content:text 是新增一个model，

    名称为`resume`  里面有三个栏位`job_id` `user_id` `content`  `:`后面定义了这些栏位的属性

  • 当要新增栏位到这个model时，rails g migration add_attachment_to_resume
    代码加上add_column :resumes, :attachment, :string
    就增加了一个string属性的attachment栏位到resume

##Interpretive
做加分题时对Rails的架构又有新的理解

  • 一般application的文件是套用到全栈中的
  • application.scss中import了bootstrap这个gem，如果要自己定义新的class和id就在这个文件里面定义，全栈的页面都可以套用
  • layouts中<%= yield %>就是其他views页面
  • 在layouts下新建一个文件，比如admin.html.erb 然后在admin的controller中套用 layout "admin"就是只在admin下使用这个layout

第一次Meetup
