5.17 学习利用碎片时间

发表于 2017-05-17

Objective

今天没有完整的时间，利用碎片时间做了一点事情

Reflective

遇到问题时会焦虑，有时候甚至会把问题夸大

Interpretive

一般周三我是很难找到完整的一段时间的，比如晚上开始学习都已经十一点了，今天走在路上我思考了一下这种情况要怎么办，然后想出了几个利用碎片时间的办法。

  • 在slack上看看同学的提问和助教的解答，了解一下自己没有遇到过的bug，自己想想debug的思路，再看看助教是怎么解决的

  • 看看同学的logdown，经常会看到一些干货分享。比如今天我看到了好几个同学都记录到自己或者队友写了完整的userstory，又提醒了自己，把自己零散记录的想法写成userstory，因为已经有记录，只用整理，花不了什么时间。还有个收获是看到有同学参考了我的分享，挺高兴的，于是跟着文志鹏师兄的分享给logdown开了个评论功能。

  • 在脱产群中经常有技术性问题的讨论，白天就那么看过去了，晚上回来可以认真从头看一看，有适合自己的也可以尝试一下或者记录下来

  • 碎片时间里也可以想想教程、知识点笔记、对教程的复盘笔记，又或者想想自己之前想要做的一些功能的开发思路是怎么样的。突然GET到锤子的“闪念胶囊” ，真的是神器啊，好想用啊！

  • 做一些用时不用那么长的事情，比如读一篇英文文章。

  • 总的来说，这些都是些用时短、不太费力的功夫

还有两个重要领悟是

  • 学习一定要跳出舒适区，切忌做一个假装勤奋者，只挑些不用思考的事情来做

  • 如果遇到情绪低落的情况，一定要及时想通，把自己拉出来（想通这个过程也能GET到不少新领悟）

Decisional

明天把切换订单状态的教程做完

【总结】关于routes

发表于 2017-05-14

#以购物车为例

#resources

当我们建一个controller时，要在routes.rb中加入一个resources :xxx

有了这个resources，最基本的7个action，都可以有正确的导向，参考教材

resoureces也可以嵌套使用，以job-listing为例

运行rake routes看一下

可以看到，这个时候resume是在job之下的，它的路径为job_resumes edit_job_resume这样

#namespace

如购物车中，我们建了一个admin/product

这时候如果直接加 resources :products 因为我们还有一个product_controller.rb rails会不知道你要指向哪个product，所以要用到namespace，指明是admin下的product

#member和collection

前面说到，resources会帮我们导向的正确的action，比如create用到的是POST

当我们新增一个action时，就要自己在routes定义这个action

  • member （成员）是“单数”，不加 s
  • collection （集合）是“复数”，加 s

比如我们在producsts_controller.rb中加入了一个新action：add_to_cart ，把一件商品加入购物车，是一个单数，所以我们用到member，他用到的method是POST

我们在carts_controller.rb中加入了一个新action：clean，而购物车的商品全部清理，是复数，所以我们用到collection，用到的method是DELETE

运行rake routes看一下

【总结】关于find和find_by

发表于 2017-05-14

###find
找id（一笔）
找到指定的对象，可以有多个选项
例如

找不到的话返回一个 RecodrNotFound 的错误

###find_by
找某一笔
需要传递一个hash作为参数

找到符合条件的第一个信息

举例，在jdstore中，删除购物车中的某个商品

找不到的话返回nill值
利用这一点，应该find_by设计购物车

当find_by找不到合适cart_id时，返回一个nill值，接下来可以做判断，如果是空值的话就create一个cart

这一段中的find_by就是通过product_id去确定要删除商品所在的cart_items

###where
找多笔

where returns an ActiveRecord::Relation (not an array, even though it behaves much like one), which is a collection of model objects. If nothing matches the conditions, it simply returns an empty relation.

大概理解，在joblisting中用过where捞出每个企业发布的职缺

有一个特别的用法
Product.where(price: 1..5000)
可以找出价格1-5000这个区间的products
